Each project below started as a question — how do I ship this faster, provision this more reliably, or see inside this system once it's live. The tools changed each time; the goal didn't.

/projects/dockerize-web-app

Dockerized web app + automated CI pipeline

operational

Containerized a Vite + React + TypeScript application using multi-stage Docker builds, then wired a GitHub Actions pipeline to build and push the image to Docker Hub on every commit — turning deployment from a manual step into a background process.

  • Multi-stage builds cut production image size by ~65%, improving cold-start time.
  • Layered caching and minimal base images reduced build resource consumption.
  • Fully automated build-and-push on every commit cut release effort by ~80%.
Docker GitHub Actions React TypeScript Vite
/projects/terraform-ec2

Infrastructure provisioning with Terraform on AWS EC2

operational

Replaced manual EC2 setup with Infrastructure-as-Code, so staging and production environments are defined once and reproduced identically every time instead of assembled by hand.

  • IaC via Terraform cut environment setup time by ~70%.
  • Consistent, repeatable deployments across staging and production.
  • Dynamic provisioning with Terraform's for_each meta-argument scales multiple EC2 instances from a single config.
Terraform AWS EC2 IaC
/projects/eks-monitoring

Cloud-native monitoring system on AWS EKS with CI/CD

operational

Built and containerized a cloud-native application, automated its build and push via GitHub Actions, then deployed it to AWS EKS with Kubernetes manifests and layered Prometheus and Grafana on top for real-time visibility.

  • Automated image build/push reduced manual build effort by ~85%.
  • EKS deployment achieved ~99.9% uptime and cut provisioning time by ~70%.
  • Prometheus + Grafana monitoring improved incident detection time by ~60%.
Kubernetes AWS EKS Prometheus Grafana GitHub Actions Docker
